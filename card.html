.NET Deep Dive
Most devs use IOptions<T> wrong.
There are 3 variants — and they behave completely differently.
01IOptions<T>Singleton
❌ Never refreshes✓ Registered once at startup
Values are frozen at app start. Safe for static config — but useless if you need live updates.
02IOptionsSnapshot<T>Scoped
✓ Refreshes per request⚡ Great for Web APIs
Re-reads config on every HTTP request. Perfect for feature toggles in multi-tenant web apps.
03IOptionsMonitor<T>🏆 Best
✓ Live reload✓ Singleton safe⚡ OnChange() callback
Real-time config change notifications. No app restart. The go-to for feature flags and hot config.
// Quick Rule of Thumb
🔒Static, never changesIOptions
🔄Changes between requestsIOptionsSnapshot
⚡Live reload, zero downtimeIOptionsMonitor
